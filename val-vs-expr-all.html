VAL vs EXPR avec ALL — divergence des modes de passage · Preview depuis sept. 2025

EXPR = Lazy VAL = Eager
Quand la différence apparaît vraiment

Modifiez la fonction pour qu'elle ignore le contexte courant avec ALL — là, VAL et EXPR divergent immédiatement.

Avec VAL + ALL
FUNCTION SommeAvecTotal = (
A : INT64 VAL,
B : INT64 VAL
) =>
CALCULATE( A + B, ALL(Membres) )
-- ALL sans effet : A+B déjà scalaires
A et B sont déjà des nombres (ex: 6 et 4). Le ALL ne peut pas les "dé-calculer". La fonction retourne 10 — la valeur de la ligne, pas le total.
Résultat : valeur de la ligne — ALL ignoré car les scalaires sont figés.
Avec EXPR + ALL
FUNCTION SommeAvecTotal = (
A : INT64 EXPR,
B : INT64 EXPR
) =>
CALCULATE( A + B, ALL(Membres) )
-- ALL agit AVANT l'évaluation de A+B
A et B sont des expressions non-évaluées. CALCULATE applique ALL, puis évalue A ([Nb_Adultes]) et B ([Nb_Enfants]) dans ce nouveau contexte sans filtre.
Résultat : total général (12) sur chaque ligne — ALL a effacé le filtre avant l'évaluation.
RÉSULTATS PAR CATÉGORIE
Catégorie VAL + ALL EXPR + ALL Écart
Adultes 6 12 ALL ignoré / ALL actif
Enfants 4 12 ALL ignoré / ALL actif
Séniors 2 12 ALL ignoré / ALL actif

La règle d'or

VAL → l'expression est évaluée avant d'entrer dans la fonction. CALCULATE interne ne peut plus la modifier.
EXPR → l'expression est évaluée à l'intérieur du contexte que la fonction construit via CALCULATE.
Identiques → quand la fonction ne modifie pas le contexte (comme A + B seul), VAL/EXPR ne divergent que si un CALCULATE, ALL, DATEADD… intervient dans le corps de la fonction.
